## 一、對話總結（6/11 → 6/14）

6/11 交付完整合約 PDF 後，客戶於簽約前再啟動一輪密集修改要求，主要分四大主題：

### 1. 主合約條款微調（6/11 深夜 01:00 起）

客戶一次提出 9 項要求：附件 D $175 補足款是否該付、純維護費結構、匯款手續費改 SHA、會議異議期 24→48hr、主觀體驗改客觀判定、逾期未驗收「視同通過」改催告制、Repo Admin 權限緊急例外、逾期付款 14 天後果（反對斷線/刪資料）、保固範圍擴大。開發方逐項回覆並起草新條文，**多數已落實到目前的 `contract-zh.md`**（見第二節）。

### 2. 數字攻防（6/12–6/13）

- **滯納金**：客戶反對 16%（稱高利貸），要求年息 5%；開發方主張 16% 為民法 205 條合法上限，後表示可退回 12%（月息 1%）。**目前 md 為月息 2%/年息 24%，以民法 250–253 條「違約金」性質重新定性以規避 205 條上限——此版本較雙方口頭共識更強硬，屬未定案。**
- **逾期付款天數**：客戶要 30 天，開發方還價 21 天。目前 md 仍為 14 天，未更新。
- **效能基準**：客戶要冷啟動 ≤2 秒、API ≤1 秒；開發方還價冷啟動 ≤3 秒、API ≤1.5 秒。目前 A-3-10 仍為 ≤4 秒 / ≤2 秒，未更新。
- **$2499 報價爭議**：客戶質疑原報價 $2000 漲到 $2499 不合理，要求拆「原始報價／第二版／維護價」三個數字。開發方守住 $1800 優惠價＋$699 折扣綁定 4 個 SOW 週期之框架，拒絕逐項拆價，理由為範疇由「沿用 MVP」變更為「AWS Lightsail 從零重建」。**爭議未完全收斂，但開發方立場未讓。**

### 3. Repo Admin 權限爭議（6/13，客戶在此回合佔上風）

客戶堅持身為 Repo 所有人，基於資安/IP 保護有權隨時收回開發方權限，且若為誤判不應承擔任何違約金。經多輪攻防，開發方讓步：**最終共識為「客戶須書面通知＋檢附證據；若為誤判，恢復權限後開發時程（含驗收與付款節點）全部順延、客戶不得異議；雙方均不負違約責任、無罰款」，且不再觸發 5.9.1。** 目前 md 的 5.4.2 仍保留「誤判 → 視為違反 5.4.1 → 違約後果」的舊版，**與此共識相反，需改寫**。

### 4. 維護費定案＋第二版（App 2.0）報價（6/13–6/14）

- **維護費**：定案 **$500/月、含 12hr**，已寫入 appendix-d.md D-5.6（含 In-Scope / Out-of-Scope / 計費三子條）。客戶仍質疑 $50/hr 偏高（稱美國可找 $40/hr），但 $500 月費本身雙方同意。
- **App 2.0**：開發方產出需求訪談書（`app-2.0-requirements-interview.md`），完整範疇估 $15,850–22,650。客戶認為超預算、且主張 AI/vibe coding 應更便宜，要求 milestone-based 計價、單月預算約 $1,000–1,200。**最新（6/14 02:33）收斂為：付費影片 MVP＋Apple Pay＋採 Cloudflare Stream 影片託管＋3 個月維護＝總報價 $3,500，OpEx（雲端用量）另計。** 客戶要求出一版含第二版內容的完整報價，開發方回覆 OK，**尚未交付。**
