## 2. 需求拆解

逐模組拆解實際需開發之元件。**粗體**為缺少將導致功能不可運行、可被盜用、或遭 App Store 退件之必要項目。

### 模組 A — 付費影片基礎設施（Admin 上傳 → 學生付費觀看）

- Admin 影片上傳：分段上傳（chunked upload）、大檔處理
- **影片轉碼 pipeline：HLS 多碼率自適應串流、縮圖產生**
- **影片儲存與 CDN 派送（AWS S3 + CloudFront 或同級）**
- 學生端播放器：HLS 自適應、斷點續看、播放進度記錄
- **付費權限門禁：Signed URL / token-based 存取控制（防止未付款抓取 URL 觀看）**
- 購買解鎖流程：權限（entitlement）資料模型、串接 Stripe 付款、解鎖狀態、存取 gating
- 影片目錄/列表：分類、列表 UI

### 模組 B — 學生練習影片（UGC，本地端處理，公開/私人）

- 本地端錄製/壓縮/裁切：裝置端（React Native）影片處理，公開/私人模式切換
- 上傳與儲存：本地端處理後仍需伺服器端儲存與派送
- **公開/私人 access control 模型（權限資料模型，非單一布林 flag）**
- 練習影片 feed/列表
- **UGC 內容審核：檢舉、下架、封鎖機制（DMCA／違法內容法律責任；Apple 對 UGC 強制要求）**

### 模組 C — 評論與對話（老師 ↔ 學生）

- 評論資料模型與 threading API
- 對話 UI：老師評論、學生回覆、雙向往返
- **推播整合：新評論/新對話事件接 Firebase（v1.0 已建推播基礎）**
- 已讀狀態、未讀紅點/badge

### 模組 D — 退款操作介面

- 學生端退款請求流程與 UI
- Admin 端退款審核/處理：串接 Stripe Refund API、退款狀態追蹤與顯示
- 購買條款文字：已含於 v1.0，本版不重複開發

### 模組 E — 進階搜尋篩選

- 後端查詢/索引：依分類/價格/老師/日期篩選、排序
- 篩選與排序 UI

### 模組 F — Apple Pay

- Apple Pay 開發 + Stripe Apple Pay 串接
- Merchant ID、憑證、網域驗證設定
- **依賴：Apple Pay 啟用依賴 Apple Developer Merchant ID/憑證，與 D-U-N-S 審核同屬前置鏈（v1.0 已知議題），排程須置於依賴解除之後**

### 模組 X — 跨模組共用

- **Admin 後台擴充：影片管理、退款審核、內容審核儀表板**
- **資料庫 schema 擴充與 migration：entitlement、評論、退款、影片 metadata**
- **AWS 基建設定：影片儲存、CDN、轉碼運算環境**
- 觀看數/觀看時長 analytics
- QA/測試/缺陷修正
- 整合、專案管理與緩衝

## 3. 非功能性風險與責任歸屬

### 3.1 OpEx — 持續營運成本（客戶承擔）

影片功能引入持續性雲端成本，非一次性開發費，由客戶長期月付：

- 影片儲存（隨影片數量累積）
- CDN 流量（隨觀看量上升）
- 轉碼運算（每支上傳影片觸發）

### 3.2 App Store / Play Store 審核風險

- UGC 影片 App 審核更嚴格，須具備檢舉、封鎖、內容過濾機制（見模組 B）
- 付費數位內容須符合各平台內購/外部支付政策
- 退件機率較 v1.0 高，平台審核緩衝期須拉長（比照主合約 4.7 緩衝機制）

### 3.3 法律/合規

- UGC 內容需檢舉與下架流程（DMCA/違法內容）
- 影片內容之隱私與授權條款須更新（沿用 v1.0 範本架構，文字另議）

## 4. 工時與費用

**費率：USD $50/hr（附件 D-5.4 國際遠端外包費率下限）　|　工作天：8 hr/day**

App 2.0 為**獨立大型開發案**，非 v1.0 之微調或延伸。其核心為影片平台（上傳、轉碼、串流、付費牆、UGC、審核），工程量級遠高於 v1.0 課程預約功能。以下為完整範疇之工時與費用。

### 4.1 完整範疇工時與費用

開發（React Native + Node.js + PostgreSQL + AWS）。含整合，QA 已內含於各模組。

| 模組 | 內容 | 工時（低–高） | 工作天（低–高） | 費用 USD（低–高） |
|---|---|---|---|---|
| A 付費影片基建 | 上傳 + HLS 轉碼 + CDN + 簽章門禁 + 播放器 + 解鎖 + 目錄 | 88–124 h | 11–15.5 天 | $4,400–$6,200 |
| B 練習影片 UGC | 本地錄製/壓縮/裁切 + 上傳 + 公私模型 + feed + 審核 | 57–81 h | 7–10 天 | $2,850–$4,050 |
| C 評論 + 對話 | 資料模型 + 對話 UI + 推播 + 已讀 | 36–48 h | 4.5–6 天 | $1,800–$2,400 |
| D 退款 | 學生請求 UI + Admin 處理 + Stripe Refund API | 18–24 h | 2.25–3 天 | $900–$1,200 |

不含 OpEx（雲端持續成本，第 3.1 節，客戶承擔）。

### 4.2 市場行情對照

同類影片 App（含影片轉碼/CDN 串流/付費牆/UGC 上傳與審核）之國際遠端外包行情約 USD $40–80/hr，整案普遍落在 **USD $25,000–$40,000 以上**。

本報價採 **$50/hr（行情下緣，等同 v1.0 附件 D 既定費率，未上調）**，**整體 $15,850–$22,650 仍低於市場行情**，非高報。

### 4.3 可選分期（降低單期現金負擔，非總價折讓）

若單期預算有限，可將高成本模組延後至附件 D 後續週期分批開發。**分期僅改變付款節奏，總工程量與總費用不因此減少。**

| 可延後模組 | 延後考量 |
|---|---|
| A 影片轉碼 pipeline（HLS 多碼率 + CDN 最佳化） | 可先以漸進式 MP4 上線，流量/畫質需求成熟後再導入 |
| B 練習影片 UGC（本地端處理、公私模型、feed、UGC 審核） | 裝置端影片處理 + UGC 審核為最大成本與法律風險來源，宜獨立週期完整開發 |
| F Apple Pay | 卡 Apple Developer Merchant ID / D-U-N-S 前置鏈，先決條件未解除前無法串接 |
