## A-3 v1.0 功能細項

### A-3-1 帳號與角色

- 支援三種登入方式：Email / 密碼、Google OAuth、Apple Sign-In（含 Face ID / Touch ID）
- 支援手機號碼獨立綁定（非登入方式，為帳號附加資訊）
- 系統角色分三級：Student（學員）、Teacher（教師）、Admin（管理員）
- 各角色 API 存取權限獨立，跨角色存取返回 403

### A-3-2 課程功能

- 課程列表顯示：名稱、時間、價格、剩餘名額；下架課程不顯示
- 課程詳情頁：完整說明、地點、名額、講師姓名及個人介紹
- 名額鎖定：名額更新採資料庫原子操作（如悲觀鎖或原子遞減），確保並發預約不超額，滿額後預約按鈕關閉
- 預約截止時限：超過截止時間後預約按鈕自動關閉
- 限時搶購（Flash Sale）：課程可設定開賣時間與結束時間，僅於該時間區間內可預約；額滿後預約按鈕關閉，與一般課程共用名額鎖定機制
- 佔位釋出：使用者建立訂單但未於 10 分鐘內完成付款者，系統自動釋出其佔用之名額，供其他使用者預約
- 防超賣與資料一致性：名額異動（扣減、釋出）以資料庫交易層級鎖定/原子操作為基礎保證，確保顯示與實際庫存一致；若上線後流量規模需額外快取層（如 Redis）以提升讀取效能，屬技術實作選項，將於附件 D 長期合作框架視實際需求另行評估導入，不影響本節所定之防超賣保證
- 取消政策（Class Pass 機制）：
  - 課程開始前 24 小時以上得取消，系統自動釋出名額，不退款，改發放 Class Pass；課程開始前 24 小時內不可取消。
  - Class Pass 適用於所有課程；一張 Class Pass 之抵用額度等同該次取消課程之原購買金額。
  - Class Pass 不可轉讓予他人；同一帳號得同時持有多張 Class Pass。
  - 取消次數限制：每一帳號每月（30 天）以「24 小時以上之取消」最多 3 次為限；同一張 Class Pass（即由取消所發放之抵用額度）不得再次取消。
  - Class Pass 自發放日起 1 個月（30 天）內有效，逾期未使用即失效，無補救或展延機制。
- 我的課程頁：已預約、已購買、歷史紀錄三分類

### A-3-3 付款流程

- 透過 Stripe Payment Intent 建立付款請求
- 訂單狀態以 Stripe Webhook 為唯一更新依據，並對 Webhook 事件進行冪等處理：使用者完成付款後即使立即斷線，後端仍能正確更新訂單狀態、扣減名額並生成 QR Code；使用者重新連線後，於「我的課程」可正常顯示已生成之票券
- 付款失敗顯示明確錯誤訊息，課程不被開通
- 已購買課程不顯示購買按鈕，防止重複付款
- 購買前顯示取消政策條款文字，使用者須勾選同意後方可付款

### A-3-4 QR Code 簽到

- 已預約學員可於課程卡片查看並出示專屬 QR Code
- QR Code 與學員帳號及課程綁定，課程開始後 15 分鐘自動失效
- 教師介面提供相機掃描入口，掃描後顯示學員姓名、課程資訊及出席狀態
- 已核銷 QR Code 再次掃描顯示「已簽到」，不重複計算
- 掃描無效或已失效 QR Code 顯示明確錯誤訊息

### A-3-5 後台管理（Web-based）

- 課程管理：新增、修改、上架、下架；上架後前台即時顯示
- 會員管理：查詢會員、指派或移除教師角色、停用帳號
- 訂單查詢：學員姓名、課程名稱、付款狀態、金額
- 出席記錄：各課程出席名單與簽到狀態

### A-3-6 推播通知

- 課程提醒：已預約課程於開始前 24 小時及 1 小時各發送一次
- 搶購提醒：使用者可對指定課程設定開搶提醒，開搶前收到推播
- 使用者可自行設定是否開啟各類推播通知

### A-3-7 公司官網 Landing Page（M-08）

- 部署於客戶現有網域 motise.net
- 內容包含：公司名稱、簡短介紹、服務說明、聯絡資訊

## A-4 技術架構

### A-4-1 系統分層

```
┌─────────────────────────────────────────┐
│         React Native App                │
│      iOS + Android 行動應用程式          │
└──────────────────┬──────────────────────┘
                   ↕ HTTPS / REST API
┌─────────────────────────────────────────┐
│      Node.js — AWS Lightsail Instance   │
│      後端 API 與商業邏輯                 │
└──────────────────┬──────────────────────┘
                   ↕
┌─────────────────────────────────────────┐
│      PostgreSQL                         │
│      AWS Lightsail Managed Database     │
│      + Lightsail Object Storage         │
└─────────────────────────────────────────┘

第三方服務整合
├── Stripe          線上付款
├── Firebase FCM    推播通知
├── Apple OAuth     Apple Sign-In
└── Google OAuth    Google 登入
```

### A-4-2 技術選型

| 技術 | 用途 | 選型理由 |
|---|---|---|
| React Native | iOS / Android App | 單一程式碼庫支援雙平台，降低長期維護成本 |
| Node.js | 後端 API 與商業邏輯 | 前後端統一 JavaScript，維護人員招募門檻低 |
| PostgreSQL | 關聯式資料庫 | 成熟穩定，Lightsail 提供託管服務免自行維運 |
| AWS Lightsail | 雲端基礎設施 | 固定月費（約 $25 USD/月），成本可預測 |
| Stripe | 線上付款 | 業界標準，合規性高，API 成熟 |
| Firebase FCM | 推播通知 | 支援 iOS / Android 雙平台，免費額度充足 |

### A-4-3 核心資料表

| 資料表 | 說明 |
|---|---|
| users | 使用者帳號、角色、手機號碼 |
| courses | 課程資料、名額、截止時間、上架狀態 |
| bookings | 預約紀錄、狀態 |
| orders | 訂單資料、Stripe Payment Intent ID、付款狀態 |
| attendance | QR Code token、核銷狀態、簽到時間 |
